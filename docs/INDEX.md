## How to Use These Documents

1. Open the relevant roadmap before beginning a development session.
2. Confirm the current version or active phase.
3. Work from the next unchecked item rather than reopening old conversations for direction.
4. Update the roadmap as tasks are completed or moved to the parking lot.
5. Record finished release work in the changelog.

Roadmaps are working documents. Vision, architecture, decisions, and design documents are reference documents and should change only when the project direction or implementation foundation changes.
